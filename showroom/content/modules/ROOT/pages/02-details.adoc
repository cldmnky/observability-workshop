== Your lab environment

Everything is pre-deployed via GitOps. You need:

* An OpenShift namespace with admin privileges (provided)
* Web browser access to {openshift_console_url}[OpenShift Console^]
* The `oc` CLI — pre-configured in your lab terminal

Pre-deployed components:

* Prometheus, Thanos, Alertmanager (user workload monitoring)
* LokiStack with S3-compatible storage
* Tempo distributed tracing backend
* OpenTelemetry Operator + sidecar collectors
* Sample microservices application

== Verify your setup

[source,bash,role="copypaste"]
----
oc whoami
oc get pods -n openshift-user-workload-monitoring
oc get pods -n openshift-logging
oc get pods -n openshift-tempo-operator
----

== Troubleshooting

[cols="2,3",options="header"]
|===
|Problem |Solution
|`oc` not found |Refresh the lab terminal browser tab
|Permission denied |Check your namespace: `oc project`
|Console unreachable |Use the URL from the lab interface: {openshift_console_url}[OpenShift Console^]
|Metrics not appearing |Check `oc get servicemonitor -n <your-namespace>`
|Logs missing in Loki |Check `oc get clusterlogforwarder -n openshift-logging`
|===
